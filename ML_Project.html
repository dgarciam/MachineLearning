Movement Analisys - Machine Learning Project

Summary

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement - a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, your goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

Creating Partitions

We proceed to create our training and testing datasets from the cleaned data above, using 70% of the data as test data.

inTrain = createDataPartition(y=training$classe, p=0.70, list=FALSE)
training.set = training[inTrain,]
testing.set = training[-inTrain,]

Creating Parallelism and Fitting our model

Since the computing of the models are lenghty proccesses, we choose to use more than one processor.

To fit the predictive model, we are using Random Forest. This algorithm is characterized by a subset of features, selected in a random and independent manner with the same distribution for each of the trees in the forest. Well be using repeated cross validation, with a k-fold of 5, repeated 5 times.

For presentation purposes, the code in the chunk below wont be evaluated, but will be loaded already computed.

cl <- makeCluster(4)
registerDoParallel(cl)
rF.model = train(classe ~ .,
                           data=training.set,
                           method="rf",
                           importance= TRUE,
                           trControl = trainControl(method = "repeatedcv", number = 5, repeats = 5)
                          )
stopCluster(cl)
saveRDS(rF.model, file = "rFModel.rds")

Well predict with the training set in order to evaluate the models accuracy as well as the in sample errors, and we see accuracy of between 99.7% and 99.8% on the training set.

Out of Sample Errors

missClass = function(values, prediction) {
    sum(prediction != values)/length(values)
}
errRate = missClass(testing.set$classe, rF.predict.test)
errRate = errRate * 100

Our selected model has an out of sample error = 0.034 % and by consequence an accuracy of 99.966 %
